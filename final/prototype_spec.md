# TripPilot 原型页面规范（Prototype Spec）

> 本文档是原型开发前的设计规范。先定"设计语言"（颜色/字阶/间距/组件），再排每页布局。
> 目标：手机端（竖屏）、浅色为主 + 深森林绿点缀、豆包式亲和对话、干净省负担、有旅行温度。
> 原型为无后端演示，走引导式 demo（用户选固定选项推进，保留输入框/语音按钮作示意）。

---

## 第一部分 · 设计语言

这一套是所有页面共用的"地基"。铁律：**任何页面的颜色、间距、字号都从下面取值，不即兴发挥。** 这是保证 9 个页面风格统一、不显粗糙的关键。

### 1.1 设计基准

| 参数 | 值 |
|---|---|
| 设备 | 手机竖屏 |
| 设计宽度 | 390px（iPhone 常见宽度） |
| 在电脑上展示 | 页面居中，显示一个手机外框，两侧留白 |
| 最小可点区域 | 44 × 44px |

### 1.2 颜色系统

**主色调是浅暖底 + 深森林绿。** 浅色让旅行显得明媚快乐，深绿做点缀让它耐看、有质感。禁止：大面积深色、霓虹、赛博、照片渐变铺满。

| 角色 | 色值 | 用在哪 |
|---|---|---|
| **主色 · 深森林绿** | `#2E6B5E` | 主按钮、选中态、关键强调、AI 头像 |
| 主色-深（按下态） | `#1F4D42` | 按钮按下、hover |
| 主色-浅（背景） | `#E8F0ED` | 选中背景、标签底、AI 气泡底 |
| **页面底色 · 暖白** | `#FAF8F4` | 整个 App 的背景（带一点暖，不是纯白，更松弛） |
| 卡片底 | `#FFFFFF` | 卡片、气泡、输入框 |
| 主文字 | `#1F2421` | 标题、正文（近黑带一点绿灰，比纯黑柔和） |
| 次要文字 | `#6B7770` | 辅助说明、时间戳、依据 |
| 占位/禁用 | `#A8B0AB` | 输入框占位符、置灰 |
| 分割线/边框 | `#EAE6DF` | 卡片描边、分割线（暖灰） |
| 暖阳点缀 | `#E8A54B` | 极少量：旅行氛围点缀、"意外发现"高亮（不滥用） |

**状态色**（仅用于场景 C 的告警分级，其他地方不用）：

| 状态 | 色值 | 含义 |
|---|---|---|
| 🔴 高 | `#D96A5B`（柔和红，不刺眼） | 景区关闭、航班取消 |
| 🟡 中 | `#E8A54B`（暖橙） | 延误、预警 |
| 🔵 低 | `#5B93D9`（柔和蓝） | 轻微提醒 |

> 配色思路：底色暖白 + 主色深绿是"温暖 + 耐看"的组合；状态色都调柔和，避免刺眼的正红正黄破坏整体的松弛感。

### 1.3 字阶

固定这几档，全局统一，不出现第 5 种字号。

| 层级 | 字号 / 字重 | 用在哪 |
|---|---|---|
| 大标题 | 24px / 700 | 页面主标题（每页最多一个） |
| 标题 | 18px / 600 | 卡片标题、分区标题 |
| 正文 | 15px / 400 | 对话气泡、正文、选项 |
| 强调正文 | 15px / 600 | 需要强调的正文 |
| 辅助 | 13px / 400 | 依据、时间戳、次要说明 |
| 标签 | 11px / 500 | 角标、标签 |

字体栈：`"PingFang SC", "Microsoft YaHei", -apple-system, sans-serif`

### 1.4 间距系统（治"挤"和"乱"的关键）

所有边距、间隔只从这几个值里取。**上次原型显得丑，一半原因是间距随手写、疏密不均。**

| Token | 值 | 用在哪 |
|---|---|---|
| xs | 4px | 图标与文字、紧凑元素 |
| sm | 8px | 标签内边距、小间隔 |
| md | 12px | 卡片内元素间隔 |
| lg | 16px | 页面左右边距、卡片内边距 |
| xl | 24px | 大区块之间 |
| xxl | 32px | 页面顶部/底部呼吸区 |

**留白铁律**：页面左右边距固定 16px；卡片之间 12px；大区块之间 24px。宁可空，不要挤。

### 1.5 圆角与阴影

| 元素 | 圆角 | 阴影 |
|---|---|---|
| 卡片 | 16px | 极淡：`0 2px 8px rgba(0,0,0,.04)` |
| 按钮 | 12px | 无 |
| 对话气泡 | 18px（贴边那角 4px） | 无 |
| 标签/胶囊 | 999px（全圆） | 无 |
| 输入框 | 24px（胶囊形） | 无 |

> 圆角统一偏大（16-18px），是"亲和、松弛"感的来源——方正硬朗会显得冷。阴影只用极淡的一层，避免脏。

### 1.6 核心组件

这几个组件会在多个页面复用，统一定义一次：

| 组件 | 样子 |
|---|---|
| **主按钮** | 深绿底、白字、圆角 12px、高 48px、居中；按下变深绿-深 |
| **次按钮** | 白底、深绿字、深绿描边、同尺寸 |
| **选项卡（引导式）** | 白底、圆角 12px、左对齐文字、点击后变主色-浅底 + 深绿描边；这是 demo 里用户"选答案"的主要方式 |
| **AI 气泡** | 主色-浅底 `#E8F0ED`、左对齐、圆角 18px（左下角 4px）、前面带一个绿色小圆头像 |
| **用户气泡** | 深绿底、白字、右对齐、圆角 18px（右下角 4px） |
| **信息卡片** | 白底、16px 圆角、极淡阴影、内边距 16px |
| **数据来源标签** | 11px、主色-浅底、深绿字、圆角全圆——标在引用外部数据处（体现"不编数字"） |
| **底部输入区** | 胶囊输入框（占位"和 TripPilot 说说…"）+ 右侧语音按钮 + 发送按钮；demo 中可点但引导用选项卡 |

> **关于输入区**：保留输入框和语音按钮，是为了让体验者知道"真实产品能自由说话打字"；但 demo 数据是固定的，所以主要用**选项卡**引导用户走标准流程，避免自由输入后得不到匹配结果的落差。

## 第二部分 · 页面清单与优先级

原型覆盖 B+C 完整流程（一趟云南旅行，从行前决策到行中应急），能从头点到尾。但不是每页平均用力——**分三档，把注意力集中在最能体现 AI 价值的核心页，其余保证干净统一。** 这本身就是 MVP 优先级判断的体现。

### 页面流程（按用户真实顺序）

```
P1 首页 ──▶ P2 陪聊需求 ──▶ P3 偏好排序 ──▶ P4 对比报告 ──▶ P5 行程确认
                                                              │
                                                （出发后进入行中）
                                                              ▼
P6 行程仪表盘（监控）──▶ P7 应急预警 ──▶ P8 Plan B 方案 ──▶ P9 执行引导
```

### 三档优先级

| 档位 | 页面 | 为什么这样定 |
|:---:|---|---|
| 🌟 **高光**（做到惊艳，交互/细节拉满） | **P2 陪聊需求**、**P4 对比报告**、**P8 Plan B 方案** | 这三页是产品核心价值的高光时刻：AI 陪你聊清纠结、给你能说服自己的对比、出事时端上可执行方案。评审的注意力应该被引到这里。 |
| ✅ **完整**（干净、统一、流程顺，不炫技） | P1 首页、P3 偏好排序、P5 行程确认、P6 行程仪表盘、P7 应急预警、P9 执行引导 | 保证整条线能顺畅走通、视觉统一。简洁本身不掉分，掉分的是风格不一致或半成品感。 |
| ⏸️ **提及**（不做，标注即可） | 长期记忆、自动订票 | 用一句"Phase 2"带过，和 PRD Roadmap 呼应，体现迭代意识。 |

### demo 关键手法

- **引导式推进**：用户通过点"选项卡"选择固定答案往下走（数据无后端，走设计好的 happy path）。
- **保留输入区**：底部保留输入框 + 语音按钮作示意，让体验者知道真实产品能自由对话，但引导其用选项卡，避免自由输入得不到匹配结果的落差。
- **一条龙可点通**：P1 → P9 能完整点完，展示"从灵感到落地、再到行中兜底"的连贯体验。
